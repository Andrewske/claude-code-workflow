# Review Suggestions

You are presenting improvement suggestions generated by Claude Conscious session analysis.
Each suggestion is backed by evidence from real session data.

## Step 1: Load and summarize

Read the pending suggestions file:

```
~/.claude/conscious/suggestions/pending.jsonl
```

If the file doesn't exist or is empty, say: "No pending suggestions. Run the analysis first: `python3 ~/.claude/conscious/scripts/analyze_sessions.py --days 7`" and stop.

Parse each line as JSON. Group suggestions by `category`. Sort within each category by priority (critical > high > medium > low).

Display a summary table:

```
PENDING SUGGESTIONS
═══════════════════════════════════════════════════
Category            Count   Highest Priority
─────────────────────────────────────────────────
permissions           N     high
claude_md_updates     N     medium
skill_ideas           N     low
─────────────────────────────────────────────────
Total: N suggestions
```

Then ask the user which category to review using AskUserQuestion. Options:
- Each category with its count (e.g., "permissions (47)")
- "All categories"

## Step 2: Present suggestions one at a time

For each suggestion in the selected category (sorted by priority descending, then evidence_count descending):

Present it clearly:

```
[N/TOTAL] [PRIORITY] CATEGORY
─────────────────────────────────────────
Title: {title}
Evidence: {description}
Proposed action: {action_type} → {action_target}
  Value: {action_value}
```

Then ask via AskUserQuestion with options:
- **Apply** — Execute this suggestion now
- **Dismiss** — Reject this suggestion (will ask for a one-line reason)
- **Defer** — Revisit later
- **Skip** — Move to next without deciding
- **Stop reviewing** — End the review session

## Step 3: Apply suggestions

### For `settings_edit` (permission rules):

Read `~/.claude/settings.json`. Parse as JSON. Navigate to `permissions.allow` array. Check if the rule (`action_value`) already exists. If not, append it. Write the file back with `json.dump(settings, f, indent=2)`.

After applying, show: "Added permission rule: `{action_value}`"

### For `edit_file` (CLAUDE.md updates):

Read the target file. If it's CLAUDE.md and the suggestion is an interaction profile update:
- Look for an existing `## Interaction Profile` section
- If found, replace its contents with the new profile lines
- If not found, append the section at the end

After applying, show the diff of what changed.

## Step 4: Track disposition

After each suggestion is processed:

- **Applied**: Append to `~/.claude/conscious/suggestions/applied.jsonl` with `"applied_at"` timestamp and `"status": "applied"`
- **Dismissed**: Ask for a one-line reason. Append to `~/.claude/conscious/suggestions/dismissed.jsonl` with `"dismissed_at"`, `"reason"`, and `"status": "dismissed"`
- **Deferred**: Append to `~/.claude/conscious/suggestions/deferred.jsonl` with `"deferred_at"` and `"status": "deferred"`
- **Skipped**: Leave in pending.jsonl (do nothing)

Remove processed suggestions (applied, dismissed, deferred — NOT skipped) from `pending.jsonl` by rewriting the file without those entries.

## Step 5: Show summary

After all suggestions in the selected category are reviewed (or the user stops):

```
REVIEW COMPLETE
═══════════════════════════════════════════════════
Applied:    N
Dismissed:  N
Deferred:   N
Skipped:    N
Remaining:  N (across all categories)
```

If any permissions were applied, remind: "New permission rules are active immediately — no restart needed."

## Staleness check

Before loading suggestions, check when the analysis was last run:

```bash
ls -lt ~/.claude/conscious/suggestions/pending.jsonl 2>/dev/null | head -1
```

If the file is older than 2 days, warn: "Suggestions are {N} days old. Run `python3 ~/.claude/conscious/scripts/analyze_sessions.py --days 7` for fresh analysis."

## Important rules

- Present suggestions ONE AT A TIME. Never batch multiple suggestions into one question.
- Always show the exact change that will be made before applying.
- For permission rules, always show the full rule string (e.g., `Bash(git push *)`).
- Never auto-apply. Every suggestion requires explicit user approval.
- If settings.json fails to parse, stop and tell the user. Don't corrupt the file.
